# Sauda AI

Сервис для мастеров и фермеров Казахстана: фотографируешь товар, отвечаешь на
три вопроса (или рассказываешь голосом) — получаешь готовую карточку для
маркетплейса. Заголовок и описание на русском и казахском, теги, рекомендуемая
цена с объяснением, готовый пост для WhatsApp и «студийное» фото товара.

Проект команды на хакатон **Tech Vision 2026**, трек «Локальная коммерция»
(Digital Economy & Future Tech). Один пользователь — одна боль: продавец из
региона делает отличный продукт, но не умеет оформить его для онлайн-продажи.

## Что работает

- **Вход по номеру телефона и 4-значному ПИНу.** Без почты, паролей и SMS:
  аудитория сервиса каждый день вводит ПИН банковской карты, а пароли забывает.
  Перебор ПИНа блокируется на сервере (5 ошибок → пауза 15 минут).
- **ИИ-карточка из фото.** Многошаговая цепочка, а не «обёртка над API»:
  1. зрение — модель извлекает из фото структуру товара (категория, материалы,
     цвета, признаки ручной работы), слова продавца важнее догадок по картинке;
  2. ценовой якорь — диапазон по категории из таблицы `price_ranges`
     (по открытым объявлениям Satu/Kaspi/OLX), чтобы модель не выдумывала цену;
  3. копирайтинг — тексты RU+KZ, теги, цена с обоснованием, пост для WhatsApp.
- **Голосовой ввод.** Продавец наговаривает описание на русском или казахском,
  Whisper расшифровывает, текст можно поправить руками.
- **«Студийное» фото.** gpt-image-1 переснимает товар: мягкий свет, чистый фон,
  сам товар не меняется.
- **Мои карточки.** Сохранение в Postgres, каждый видит только свои (RLS).

## Стек и архитектура

```
Браузер: React 19 + Vite + TypeScript + Tailwind v4 + framer-motion
  │  @supabase/supabase-js: сессия, Storage, таблица cards
  ▼
Supabase
  ├─ Auth        — пользователи (телефон+ПИН через технический email)
  ├─ Postgres    — cards (RLS), price_ranges, login_attempts, app_secrets
  ├─ Storage     — product-photos, generated-images (приватные, по подписанным ссылкам)
  └─ Edge Functions (Deno)
       ├─ auth-phone-pin — регистрация/вход, защита от перебора ПИНа
       └─ generate-card  — ИИ-цепочка: OpenAI (gpt-4o-mini, whisper-1, gpt-image-1)
```

Ключ OpenAI живёт только на сервере (таблица `app_secrets`, из браузера
недоступна). В браузер попадает лишь публичный anon-ключ Supabase — доступ к
данным ограничивает RLS.

## Как запустить локально

```bash
# 1. Зависимости (один раз)
npm install

# 2. Переменные окружения (один раз)
#    скопируйте .env.example в .env.local и подставьте URL и anon-ключ
#    своего проекта Supabase

# 3. Запуск
npm run dev
```

Vite покажет адрес (обычно http://localhost:5173). Сборка: `npm run build`.

Бэкенд уже развёрнут в облаке Supabase: миграции лежат в истории проекта,
код Edge Functions — в `supabase/functions/`. Для своего инстанса нужно:
применить миграции, задеплоить обе функции и положить ключ OpenAI в
`app_secrets` (`name = 'OPENAI_API_KEY'`).

## Структура проекта

```
src/
  content.ts                  — все тексты интерфейса на русском и казахском
  App.tsx                     — маршруты и защита приватной зоны
  index.css                   — Tailwind + фирменные цвета и шрифты
  lib/
    supabase.ts               — клиент Supabase
    auth.ts                   — телефон+ПИН: нормализация номера, вызовы auth-функции
    api.ts                    — Storage, generate-card, таблица cards
  context/
    AppContext.tsx            — контексты языка (RU/KZ) и сессии
  components/
    Navbar, Hero, ProblemSection, HowItWorks, Features,
    BeforeAfter, Audience, FAQ, CTASection, Footer   — секции лендинга
    RegisterPage, LoginPage   — вход по телефону и ПИНу
    Dashboard                 — одна большая кнопка «Создать карточку»
    CreateCardWizard          — фото → три вопроса/голос → генерация
    CardResult                — карточка, цена, пост, студийное фото, сохранение
    MyCards                   — сохранённые карточки
    AppHeader, Field, FadeIn  — общие элементы
supabase/functions/
  auth-phone-pin/             — вход/регистрация, блокировка перебора
  generate-card/              — ИИ-цепочка (карточка, голос, фото)
```

## Дизайн-система

| Роль                    | Цвет      |
| ----------------------- | --------- |
| Фон (кремовый)          | `#FAF7F2` |
| Текст                   | `#211E1B` |
| Акцент (терракота)      | `#C75B39` |
| Второй акцент (зелёный) | `#2F5D50` |
| Карточки (бежевый)      | `#F1EADF` |
| Подсветка (жёлтый)      | `#E8B04B` |

Шрифты Manrope + Inter. Интерфейс рассчитан на людей, которым мелкие кнопки
и запутанные вкладки неудобны: крупные элементы, один сценарий, два поля на вход.
